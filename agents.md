# agents.md — Agent guidance for cx-api-docs

Read `CLAUDE.md` first. This file adds agent-specific rules on top of it.

---

## Version policy

**Only v5 receives new content.** v3 and v4 are frozen for backward compatibility. Never add new services, overviews, or intro table rows to v3 or v4.

## What agents must never do

- **Never edit files under `api-reference/`** — they are fully regenerated by `mintlify-scrape` on every `make build`. Any manual edits will be overwritten.
- **Never edit `docs.json`** — it is regenerated by `build_navigation_file.py`. Edit the script or the source files instead.
- **Never modify `openapi_v{3,4,5}.yaml` directly** — these are auto-synced from openapi-facade. Edits here will be clobbered on the next sync.
- **Never add to `V3_SERVICES` or `V4_SERVICES`** — those dicts are frozen. New services go into `V5_SERVICES` only.
- **Never edit `introduction-v3.mdx` or `introduction-v4.mdx`** to add new services — those intro pages are frozen.

## What agents own

| Task | File(s) to change |
|---|---|
| Add/remove a service from the nav | `build_navigation_file.py` → `V{3,4,5}_SERVICES` dict |
| Add/update a service overview | `service-overviews/{service}-overview.mdx` |
| Add/update the landing page table | `introduction-v{3,4,5}.mdx` |

## Workflow for adding a service to docs

All additions target v5 only.

```
1. Edit build_navigation_file.py  — add entry to V5_SERVICES
2. Create/edit service-overviews/{service}-overview.mdx
3. Edit introduction-v5.mdx  — add row to the API Overview table
4. Run: make build
5. Run: mintlify dev  — visually verify the service appears and overview renders
```

Do all four file changes in a single commit. Running `make build` regenerates `api-reference/` and `docs.json`; commit those generated changes together with the source edits.

## Verifying changes

After `make build`, check:
- The service directory exists under `api-reference/{version}/{service}/`
- `overview.mdx` is present in that directory (confirms the overview copy step ran)
- `docs.json` contains the service under the right version's navigation groups
- `mintlify dev` shows the service in the sidebar and the overview page loads

## Identifying hidden services

To find services that have generated files but are not yet in the nav:

```bash
# Lists directories in api-reference/v5/ that are absent from V5_SERVICES
comm -23 \
  <(ls api-reference/v5/ | sort) \
  <(python3 -c "from build_navigation_file import V5_SERVICES; [print(k) for k in sorted(V5_SERVICES)]")
```

## Overview file format

Follow the pattern of existing files in `service-overviews/`. Minimum sections:

```markdown
# {Service display name} overview

{One paragraph describing what the service does.}

## Authentication and permissions

{API key requirements and permission table.}

## Common error response codes

{Table of status codes.}
```

Do not add version-specific content to overview files — they are copied verbatim into all versions that have the service.

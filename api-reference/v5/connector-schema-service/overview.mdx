# Connector Schema service overview

The Connector Schema service returns the schema definition for a Notification Center connector or preset of a given destination type. The schema lists the fields required to set up the connector or preset, including each field's name, type, allowed values, and whether it is required. Call this service before creating a connector or preset so you know which fields to provide.

Learn more in our [documentation](https://coralogix.com/docs/user-guides/notification-center/introduction/).

## Authentication and permissions

To use the Connector Schema service API you need to [create a personal or team API key](https://coralogix.com/docs/user-guides/account-management/api-keys/api-keys/). Add the Notification Center permission below to the key.

| Presets | Permission            | Description                                      |
|---------|-----------------------|--------------------------------------------------|
| NotificationCenter | `NOTIFICATION-CENTER-CONNECTORS:READCONFIG` | View connectors in the Notification Center and their field data. |

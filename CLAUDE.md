# cx-api-docs

Mintlify-based API documentation built from OpenAPI specs.

## File ownership

| Path | Owner | Notes |
|---|---|---|
| `openapi_v{3,4,5}.yaml` | CI (auto-synced from openapi-facade) | Never edit manually |
| `api-reference/{v3,v4,v5}/` | Generated by `mintlify-scrape` | Never edit manually |
| `docs.json` | Mintlify config; `navigation.versions` regenerated by `build_navigation_file.py` | Edit freely except `navigation.versions` |
| `service-overviews/{service}-overview.mdx` | Human-owned | One file per service |
| `introduction-v{3,4,5}.mdx` | Human-owned | Version landing pages |
| `build_navigation_file.py` | Checked-in script | Edit to add services to nav |

## Version policy

**v5 only for new content.** v3 and v4 are frozen for backward compatibility — no new services, overviews, or intro rows. Bug fixes to existing content only.

## What `make build` does

1. `mintlify-scrape` reads each OpenAPI spec → writes per-operation `.mdx` files into `api-reference/{version}/{service}/` (full regeneration).
2. `build_navigation_file.py`:
   - Copies `service-overviews/{service}-overview.mdx` → `api-reference/{version}/{service}/overview.mdx`
   - Rewrites `docs.json` based on services present on disk **and** listed in `V5_SERVICES` (or `V3/V4_SERVICES`)

**Not touched by `make build`:** `introduction-v{3,4,5}.mdx`, `service-overviews/`. Those are maintained manually.

A service in the OpenAPI spec is not enough — it must also be in `V5_SERVICES` in `build_navigation_file.py` to appear in the nav.

## Adding a service to docs (v5 only)

For a service already in the spec (files exist under `api-reference/v5/`):

1. Add to `V5_SERVICES` in `build_navigation_file.py`
2. Create `service-overviews/{service}-overview.mdx` (auth table, error codes, protocol docs)
3. Add a row to the API Overview table in `introduction-v5.mdx`
4. Run `make build` and `mintlify dev` to verify

For a brand-new service not yet in the spec: get the spec merged in openapi-facade first (CI auto-syncs it here), then follow the steps above.

## Hidden services (in spec, not in nav)

These v5 services have generated files but are excluded from the nav pending overview authoring:
`api-keys-admin-service`, `cases-notification-service`, `cases-service`, `users-management-service`

## Running locally

```bash
make build    # regenerate api-reference/ and docs.json
mintlify dev  # local preview
```
